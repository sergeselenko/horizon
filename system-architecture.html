The Session Lifecycle
How every session flows from startup to closeout, and the slash commands that power it.
Session FlowThe standard progression of a Claude session from first message to last

📦
/closeout
Keep/delete, Session Log, Continuity Check.
✅
/tasks → /capture
Tasks to Notion, then instruction improvements.
Slash CommandsText-based workflow triggers — type the command, Claude reads a Notion prompt page and follows the procedure
/closeout▸
Close out a session. Session Log entry, Continuity Check.
When: End of any substantive session
Produces: Session Log entry, Chat Map registration, snapshot/digest currency check
/tasks▸
Scan conversation for tasks to capture to Notion.
When: After /closeout
Rule: Each command runs independently
/capture▸
Harvest instruction improvements, decision records.
When: After /tasks — the learning loop
Searches for: /flag markers from mid-session corrections
/emit▸
Cross-project state propagation. Broadcasts to shared surfaces.
When: Mid-session when important things happen
Writes to: Global Operator Profile + Secretary Pad
/digest▸
Update a venture's Knowledge Digest with new domain intelligence.
When: Session surfaces competitive data, pricing, people observations
/snapshot▸
Lightweight mid-session Project Snapshot update.
When: Want to save state without full closeout
/flag▸
Mid-session course correction bookmark.
When: Claude goes off-track — don't stop, just bookmark
Example: /flag — ran 5 searches before reading the obvious page
/scope▸
Lock session scope. Claude flags drift.
Expand: “Add [X] to scope” at any time
Optional: Sessions without it work normally
