Section 02
The Session Lifecycle
How every session flows from startup to closeout, and the slash commands that power it.
📖
Startup Reads
Claude reads 5-6 Notion pages via MCP before first response. ~20-40K tokens.
📋
Auto-Registration
Creates Session Log entry with status "In Progress" so tracker shows open sessions.
⚡
Work
Substantive work with Plan → Approve → Execute → Confirm gates. Mid-session: /flag, /scope, /emit, /snapshot, /digest.
📦
/closeout
Recommend keep/delete. Update Session Log. Continuity Check (snapshot, digest, engagement docs).
✅
/tasks → /capture
Capture tasks to Notion, then harvest instruction improvements and decision records.
Slash Commands
/closeout
Close out a session. Recommend keep/delete, Session Log entry, Continuity Check.
When: End of any substantive session
Reads: Prompt page in Notion
Produces: Session Log entry, Chat Map registration, snapshot/digest currency check
/tasks
Scan conversation for tasks to capture to Notion.
When: After /closeout
Rule: Each command runs independently — never auto-chains
/capture
Harvest instruction improvements, decision records, workflow docs.
When: After /tasks — the learning loop
Produces: Improvement Log entries, decision records, workflow documentation
Searches for: /flag markers from mid-session corrections
/emit
Cross-project state propagation. Extracts what matters, writes to shared surfaces.
When: Mid-session when important things happen that other projects should know
Writes to: Global Operator Profile → Recent Cross-Venture Activity + Secretary Pad → Activity Log
Optional: Venture Posture Table updates if posture changed
/digest
Update a venture's Knowledge Digest with new domain intelligence.
When: Session surfaces competitive data, pricing, people observations, strategic patterns
Pattern: MCP-native, same lightweight pattern as /snapshot
/snapshot
Lightweight mid-session Project Snapshot update.
When: Want to save state without full closeout
Scoped to: Project Snapshot page only
/flag — [desc]
Mid-session course correction bookmark. Don't stop, just bookmark.
When: Claude goes off-track
Harvested by: /capture (Category 1)
Example: /flag — ran 5 searches before reading the obvious page
/scope — [desc]
Lock session scope. Claude flags drift outside declared boundaries.
When: Want to stay focused
Expand scope: "Add [X] to scope" at any time
Scope is optional: Sessions without it work normally
